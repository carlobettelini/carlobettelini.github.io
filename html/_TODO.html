Goal

The goal of this exercise is to create your personal website using HTML5 for the structure and content and CSS for styling.

Your website should have at least the following pages:

  1. a home page
  2. a blog page
  3. a picture gallery page
  4. a picture upload form page.

To get started, you can use the skeleton files we provide in the gitlab classroom project.

Publishing your website

Your project solution should be commited and pushed to your github class repository. Make sure your name is written in every file you submit. Anonymous solutions will not be graded.

Once complete, your website should be uploaded to the atelier.inf.usi.ch server (if you have problems with the ssh authentication using your USI account you can contact sistema.lu@usi.ch).

To do so, upload your finished website on atelier.inf.usi.ch under your /home/<your_username>/public_html/wa1 directory. If, for example, your username is johndoe, then you should put your files in /home/johndoe/public_html/wa1. We should then be able to access your website using the following URL: http://atelier.inf.usi.ch/~johndoe/wa1.

Hint: to upload your homepage on the server you can clone your git class repository there, or use rsync from your computer.

Task 1. Holy grail layout

Time to get started with the basic structure of your website which will be none other than the classic holy grail layout with a responsive twist.

Content Structure

The holy grail layout consists of a header, a navigation section, a sidebar, the main content and a footer section. Your markup should be semantically correct (for example using the <header> tag for the HEADER section).

1. Header

The header should display the name of your webpage. Feel free to put a logo if you have one.

2. Navigation

Links to all the pages of your site. Hovering on a link should transition the default background the link to a color of your liking within 250 milliseconds.

3. Main Content

This depends on the page (see below).

4. Sidebar

The sidebar should contain links to your social media profiles.

5. Footer

Additional information like copyright, the date when the page was last updated and links to legal disclaimers.

Styling and Layout

Your site should use a consistent style shared by all the pages. If you need to customize the layout/formatting of individual pages you can also add CSS rules dedicated to each page.

Use flexbox to implement this layout for all your pages. Pay attention to the design mockups provided.

For mobile-size viewports (less than than 768 pixels), your design should look like shown below:

The mobile version should satisfy the following requirements:

  • All sections should fill horizontally the visible viewport.
  • The NAVIGATION and the SIDEBAR are positioned above and below the MAIN CONTENT respectively.

Desktop

For desktop-size viewports (equal or more than 768 pixels), your design should look like shown below:

The desktop version should satisfy the following requirements:

  • The content of the page should horizontally fill the visible viewport.
  • The height of the NAVIGATION and SIDEBAR is expected to stretch all the way to the FOOTER even when there's no content inside them.
  • The FOOTER is supposed be "sticky" to the bottom of the page. This means that even if there's not enough content in the MAIN CONTENT section of your page to push the footer the bottom of the window, the FOOTER should still stick to the bottom. However, if the main content does not fit in the browser viewport, the FOOTER should be pushed out of view (the user will need to scroll to find at the bottom of the page).

Both the desktop and mobile layouts should be implemented using CSS only. Use the same HTML markup for both layouts.

Task 2. Home page

Main Content

  • A couple of paragraphs about your personal website. Feel free to list your hobbies and anything you want to share with the world.
  • Some multimedia, pick at least one among:
    • A video you like from youtube.com embedded using an <iframe> element.
    • Your favourite mp3 so that people visiting your website can listen to it, embedded with an <audio> element

Styling and Layout

The homepage text, images, audio and iframes should not overflow horizontally.

Task 3. Blog page

Your blog page should contain at minimum 3 posts, at least one of which should list you as the author.

Main Content

Each post should:

  • have a heading with the title of the post
  • have the author name and the date the article was posted right under the heading. The style for the author name and date should be different than the heading (usually a small font in an unobtrusive color). If the author of the post is you, your name should link to the homepage of your website and have a different style than the rest of the authors (you can use your default style for links if you want).
  • At least one image pertinent to the post. The markup of the image should be before the text of the post. All images for blog posts should be stored under the <your project root>/images. directory.
  • Clicking the images of the post should open a new window (tab) that displays the clicked image in full size.
  • The text of the post describing the image. There should be at least 2 paragraphs with more than 200 words in total. The text should wrap around the image. If you do not know what to write, try this Lorem Ipsum Generator

Task 4. Picture Gallery page

This page should list your favourite images.

Some requirements about the images:

  • There should be at least 8 different images. You can reuse some of the images of the blog.
  • Store all your images under the <your project root>/images. directory. All of them should be wider than 1024 pixels and taller than 768 pixels. At least three of them need to have unique aspect ratios compared to the rest of the images.

The gallery page should:

  • display all your images as thumbnails in a grid view.
  • Each thumbnail should be no larger than 200 pixels and no shorter than 80 pixels in both dimensions.
  • Each thumbnail should display the image in the original aspect ratio.
  • There should be whitespace between thumbnails.
  • If there's not enough horizontal space to fill all the thumbnails they should wrap to the next line instead of horizontally overflowing the MAIN CONTENT section.

Task 5. Picture Upload Form

Main Content

The page should contain a form with the following input fields:

  • a field for the picture title
  • an email field for the picture author
  • a multi-line field for the optional picture description
  • a field to the upload of the image file
  • a field to denote whether the image should be marked as favourite or not
  • a slider to rate the quality of the image (from 0 to 10)
  • the submit button

Every field should have a label that denotes what this field expects as a value. Opt for a two-column layout where on the left you have the labels and on the right the fields. The submit button doesn't need a label since it can display the "Submit" text within its markup.

Pay extra attention to using the correct fields, buttons and attributes for the form. For example, if the user tries to submit the form with an email that has invalid format, the form should show an error message. Using the correct configuration, allows you to do this with plain markup.

Task 6. Optional Features

  1. Navigation

    Highlight the navigation entry corresponding to the currently active page.

  2. Custom Fonts

    Your site may optionally also load and use at least one custom font. Feel free to either use Web fonts, or to specify one using @font-face CSS rule.

  3. Interactive Picture Gallery

    • Smoothly change the style of the thumbnail image on which the mouse hovers. For example, make the image larger, add a shadow or a colored frame, use a CSS Filter.
    • Clicking a thumbnail should open a new browser window (or tab) that displays the image in full size.
    • Hovering on a thumbnail should display the pointer cursor.

  4. Mobile Phone Camera

    In the picture upload form, connect the image file field to the phone camera with HTML Media Capture

You should update the README.md file enumerating the optional features you implemented.
